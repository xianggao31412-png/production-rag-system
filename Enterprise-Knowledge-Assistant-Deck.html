Why most RAG demos fall short

02 · What it is

The idea in one line

It reads your files and answers
with the source attached.

Drop in TXT, Markdown, CSV, or PDF. The platform indexes every passage with both keyword and vector search, then answers natural-language questions — each claim backed by a numbered citation, so you always know where it came from.

It runs fully offline by default, and upgrades to a local large language model by changing a few settings.

05 · Hybrid retrieval

Two signals, fused

Vector · semantic

Understands meaning

Finds passages about the same idea even when the wording differs — "time off" matches "annual leave".

RRF FUSION
BM25 · lexical

Nails exact terms

Matches the precise tokens that embeddings blur — product codes, acronyms, names, and IDs.

Reciprocal Rank Fusion combines them by rank, not raw score — robust across retrievers and needs no per-corpus tuning.

06 · Grounded & reliable

Answers you can trust

Every claim is cited.
It never hard-fails.

Each answer carries inline numbered citations and a grounded flag. When no language model is reachable, an extractive engine assembles the answer only from real passages in your documents — so it degrades gracefully instead of breaking, and can never fabricate facts.

09 · Deployment

Runs anywhere — scales up by config

Offline by default
  • No downloads, no GPU, no network. Boots in seconds.
  • Deterministic hashing embedder + in-memory vector store.
  • Extractive answer engine — always grounded and cited.
  • One-click launcher and a guided showcase demo.
Production profile
  • Local Ollama — qwen3-coder for answers, nomic-embed for vectors.
  • ChromaDB vector store for larger corpora.
  • Cross-encoder reranker for top-tier relevance.
  • Docker image and Compose, switched by environment variables.

10 · Engineering rigor

Built and verified in iterations

Integration testing found real bugs — and fixed them.

~40 tests

A unit and integration suite covering chunking, retrieval, fusion, reranking, storage, and the full API.

Bug · BM25

Small-corpus negative IDF silently disabled keyword search; fixed to rank by lexical overlap instead of score sign.

Bug · Logging

A reserved log field crashed every upload; hardened so a reserved key can never crash a request again.

Bug · Readiness

A property called like a method reported a healthy component as down; corrected and re-verified.
